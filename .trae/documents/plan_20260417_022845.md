# 将协同过滤升级为混合推荐系统（协同过滤 + 基于内容推荐）

## 现状分析
当前推荐系统仅使用基于杰卡德相似度的协同过滤算法，缺乏对景点内容特征（城市、类型、评分等）的利用。

## 实现计划

### 1. 实现基于内容的推荐算法
- 创建 `app/services/content_based_service.py` 文件
- 实现景点特征提取（城市、类型、评分等）
- 计算用户偏好特征向量
- 实现基于余弦相似度的内容推荐

### 2. 实现混合推荐策略
- 创建 `app/services/hybrid_recommender.py` 文件
- 实现混合推荐类，结合协同过滤和基于内容推荐
- 支持可配置的权重分配（协同过滤权重、内容推荐权重）
- 实现多种混合策略（加权混合、切换混合、分段混合）

### 3. 更新推荐接口
- 修改 `app/services/neo4j_service.py`，保留原有协同过滤接口
- 添加 `recommend_by_hybrid` 函数作为新的推荐入口
- 更新 `app/api/recommend.py`，添加混合推荐端点 `/recommend/hybrid`

### 4. 增强可解释性
- 为混合推荐结果提供详细的推荐理由
- 说明推荐来源（协同过滤/基于内容）
- 结合两种算法的优势提供更有说服力的推荐

## 技术实现细节

### 基于内容的推荐
- 提取景点特征：城市（one-hot）、类型（one-hot）、评分（归一化）
- 构建用户偏好向量：基于用户历史足迹的特征加权平均
- 计算余弦相似度进行推荐

### 混合推荐策略
- **加权混合**：`score = α * cf_score + (1-α) * content_score`
- 支持动态调整权重
- 提供冷启动处理（新用户/新景点）

## 预期效果
- 提高推荐准确性，特别是对于用户历史数据较少的场景
- 增强推荐多样性，避免推荐结果过于集中
- 提升推荐系统的可解释性
